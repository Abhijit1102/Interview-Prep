Types of UML Diagrams

UML diagrams split into two fundamental categories depending on whether they capture the shape or the behavior of a system.

Structural (Static) Static

Describe how the system is organized at any given moment â€” the skeleton of your application.

Examples: Class Diagram, Object Diagram, Component Diagram, Package Diagram, Deployment Diagram

Behavioral (Dynamic) Dynamic

Describe how the system behaves over time â€” the pulse of your application.

Examples: Sequence Diagram, Use Case Diagram, Activity Diagram, State Diagram, Communication Diagram

Sequence Diagrams

A Sequence Diagram shows how objects interact with each other over time. It focuses on the ordering of messages â€” who sends what, to whom, and in what sequence.

Components

ComponentDescription
Objects Participants in the interaction (e.g., User, Server, Database). Displayed as labeled boxes at the top.
Lifeline A vertical dashed line descending from each object, representing its existence over time.
Activation Bar A solid vertical rectangle on the lifeline showing an object is actively processing a message.
Messages Horizontal arrows between lifelines representing method calls, returns, or signals.
Guard Conditions Bracketed conditions like [balance > 0] that control message flow.
Alt / Loop Fragments Combined fragments that model branching (alt), looping (loop), or optional (opt) behavior.

Flow Breakdown

01
Card Insertion & Validation Synchronous
Customer inserts card. ATM sends validateCard() to BankServer, which cross-checks against the Database. Returns cardValid or ejects on failure.
02
PIN Entry & Authentication Synchronous
ATM prompts for PIN. Entered PIN is forwarded to BankServer, which matches the stored hash in the Database. Returns authSuccess or increments failed attempt counter.
03
Transaction Request Synchronous
Customer selects withdrawal amount from menu. ATM sends requestFunds(amount) to BankServer, triggering a balance check in the Database.
04
Balance Check & Deduction Synchronous
BankServer verifies sufficient funds, atomically deducts the amount, and confirms the transaction. Responds with approveDispense to ATM.
05
Cash Dispensing & Session End Asynchronous
ATM dispenses cash and asynchronously prints the receipt (caller does not wait). Card is ejected and the session terminates.
